# Database Schema

PostgreSQL database managed by **Supabase**. All tables live in the `public` schema with Row-Level Security (RLS) enabled. Migrations are located in `supabase/migrations/` and are applied in numerical order.

---

## Extensions (001)

| Extension | Purpose |
|---|---|
| `uuid-ossp` | `gen_random_uuid()` fallback for UUID generation |
| `pg_trgm` | Trigram-based fuzzy text search (trending query matching) |
| `unaccent` | Accent-insensitive search (Spanish, Portuguese, etc.) |

---

## Core Tables (002)

### `chats`

Primary table for all conversation sessions.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | `text` | PRIMARY KEY | Client-generated ID |
| `user_id` | `text` | NOT NULL | Supabase `auth.uid()::text` |
| `title` | `text` | NOT NULL | Auto-generated by title-generator agent |
| `visibility` | `text` | NOT NULL, default `'private'` | `'public'` \| `'private'` |
| `created_at` | `timestamptz` | NOT NULL, default `now()` | |

**RLS Policies**

| Policy | Operation | Rule |
|---|---|---|
| `chats_select_own_or_public` | SELECT | `auth.uid()::text = user_id OR visibility = 'public'` |
| `chats_insert_own` | INSERT | `auth.uid()::text = user_id` |
| `chats_update_own` | UPDATE | `auth.uid()::text = user_id` |
| `chats_delete_own` | DELETE | `auth.uid()::text = user_id` |

**Grants**: `authenticated` (select, insert, update, delete), `anon` (select)

## Search Analytics (007)

### `search_events`

Append-only event log of every search query. No UPDATE or DELETE policies.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | `uuid` | PRIMARY KEY, default `gen_random_uuid()` | |
| `user_id` | `uuid` | FK → `auth.users(id)` SET NULL | Nullable for anonymous |
| `query` | `text` | NOT NULL | Raw query string |
| `chat_id` | `text` | FK → `chats(id)` SET NULL | Linked chat if applicable |
| `topic_ids` | `uuid[]` | NOT NULL, default `'{}'` | AI-assigned topic IDs |
| `providers_used` | `text[]` | NOT NULL, default `'{}'` | e.g. `['tavily','brave']` |
| `result_count` | `integer` | | Number of results returned |
| `latency_ms` | `integer` | | Search latency in ms |
| `country_code` | `text` | | ISO 3166-1 origin country |
| `platform` | `text` | | User agent / device context |
| `has_engagement` | `boolean` | NOT NULL, default `false` | Did user click a result? |
| `created_at` | `timestamptz` | NOT NULL, default `now()` | |

**Indexes**
- `search_events_query_trgm_idx` — GIN trigram on `query` (fuzzy aggregation)
- `search_events_created_at_idx` — `(created_at DESC)` (time-range queries)
- `search_events_user_idx` — `(user_id)`
- `search_events_topics_gin_idx` — GIN on `topic_ids`

**RLS Policies**
- SELECT: own events only
- INSERT: anyone (including anon)
- No UPDATE / DELETE (append-only integrity)

**Grants**: `anon` (insert), `authenticated` (select, insert)

## Entity Relationship Summary

```
auth.users
  │
  ├── user_profiles (1:1)
  │
  ├── chats (1:N)
  │     ├── messages (1:N)
  │     │     └── parts (1:N)          ← AI message content
  │     └── chat_topics (N:M) ──────── topics (hierarchical tree)
  │
  ├── collections (1:N)
  │     └── bookmarks (1:N)
  │           └── → chats (optional FK)
  │
  ├── search_events (1:N, append-only)
  │
  └── alert_subscriptions (1:N)
            └── → topics (optional FK)

sources (admin-managed)
  └── source_topics (N:M) ──────────── topics

trending_queries (aggregated)
  └── → topics (optional FK)
```
